# Awesome DESIGN.md

将 DESIGN.md 文件复制到你的项目中，告诉你的 AI 助手“为我构建一个像这样的页面”，就能获得像素级完美的、真正匹配的用户界面。

## 什么是 DESIGN.md？

[DESIGN.md](https://stitch.withgoogle.com/docs/design-md/overview/) 是 Google Stitch 引入的一个新概念。它是一个纯文本设计系统文档，供 AI 助手读取以生成一致的用户界面。

它只是一个 Markdown 文件。不需要导出 Figma，不需要 JSON 模式，也不需要特殊工具。将其放入项目根目录，任何 AI 编码助手或 Google Stitch 都能立即理解你的 UI 应该是什么样子。Markdown 是 LLM 阅读效果最好的格式，因此无需解析或配置。

| 文件 | 谁读取它 | 它定义了什么 |
|------|-------------|-----------------|
| `AGENTS.md` | 编码助手 | 如何构建项目 |
| `DESIGN.md` | 设计助手 | 项目的外观和感觉应该如何 |

**此仓库提供了从真实网站提取的、即拿即用的 DESIGN.md 文件。**

## 请求一个 DESIGN.md

你可以[request a DESIGN.md](https://getdesign.md/request)，指定特定网站，包括仅为你提供的私有请求。

## 每个 DESIGN.md 包含什么

每个文件都遵循 [Stitch DESIGN.md format](https://stitch.withgoogle.com/docs/design-md/format/)，并包含扩展部分：

| # | 部分 | 内容 |
|---|---------|-----------------|
| 1 | 视觉主题与氛围 | 格调、密度、设计理念 |
| 2 | 调色板与角色 | 语义名称 + 十六进制值 + 功能角色 |
| 3 | 排版规则 | 字体家族，完整层级表格 |
| 4 | 组件样式 | 按钮、卡片、输入框、导航及其状态 |
| 5 | 布局原则 | 间距标尺、网格、留白理念 |
| 6 | 深度与层级 | 阴影系统，表面层级 |
| 7 | 注意事项 | 设计护栏和反模式 |
| 8 | 响应式行为 | 断点、触摸目标、折叠策略 |
| 9 | 代理提示指南 | 快速颜色参考，即拿即用的提示词 |

每个站点包含：

| 文件 | 用途 |
|------|---------|
| `DESIGN.md` | 设计系统（供代理读取） |
| `preview.html` | 可视化目录，显示色板、字体标尺、按钮、卡片 |
| `preview-dark.html` | 相同目录，使用深色表面 |

### 如何使用

1. 将某个网站的 `DESIGN.md` 文件复制到你的项目根目录
2. 告诉你的 AI 助手使用它。
